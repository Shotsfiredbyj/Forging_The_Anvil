# Phase 4 (tier_v2) — Tier Test Analysis

## Setup

- **Winner combo**: qwen3.5-9b + qwen3.5-27b (from Phase 3 v2 cross-family leaderboard)
- **Reviewer**: mistral (cross-family, low variance)
- **Tiers**: strong, mid, weak (cached input stages at 3 quality levels)
- **Runs**: 1 cascade per tier = 3 total
- **Runtime**: ~2 hours total

## Raw results

| Tier | Grade | Pass | Task Count | A | B | C | D | Time |
|---|---|---|---|---|---|---|---|---|
| strong | **B** | 9/9 | 9 | 0 | 9 | 0 | 0 | 24 min |
| mid | **B** | 9/10 | 10 | 0 | 9 | 1 | 0 | 55 min |
| weak | **B** | 8/10 | 10 | 0 | 8 | 1 | 1 | 42 min |

## Findings

### 1. Overall grade is flat

All three tiers graded B. No letter-grade variance from strong → weak upstream quality. On a pure "overall grade" basis, the pipeline is input-quality-insensitive.

### 2. File-level gradient IS real

Pass rate does degrade with weaker inputs:
- strong: 100% pass (9/9)
- mid: 90% pass (9/10)
- weak: 80% pass (8/10)

That's a 20pp drop. Individual files fail more as input quality degrades:
- strong: zero failures
- mid: 1 file scored C
- weak: 1 file scored C, 1 file scored D

### 3. Zero A grades anywhere

Notable: the winner combo got A/B/A/A in Phase 3 v2 with mistral reviewer on the Phase 3 v2 cached stages. In Phase 4 with the tier-specific cached stages, mistral gives B/B/B.

Possible explanations:
1. **Phase 4 tier cached stages are genuinely harder content** — the mid/weak tiers have 10 tasks (not 9), suggesting more complex blueprints
2. **Single-run variance** — only one cascade per tier, and mistral's v2 distribution was {A:3, B:3}, so a single B is within the usual spread
3. **Mistral grading more conservatively on Phase 4 blueprint format** — unlikely but possible

Without a second run per tier we can't distinguish variance from systematic difference. The flat-B pattern across all 3 tiers argues for a systematic effect (content complexity) rather than pure variance.

### 4. Mid tier was the longest-running (55 min vs 24/42)

Mid tier spent ~2x as long as strong. This suggests mid-quality input triggered more rewrite cycles in Stage 5 iterative mode (review → revise → review → revise → pass). Strong inputs let cascades converge faster; mid inputs require more iterations to stabilise.

Weak (42 min) was shorter than mid — somewhat surprising. Possibly because weak inputs produce obvious failures quickly (early termination on low scores) rather than borderline cases that keep iterating.

### 5. Time is NOT monotonic with input quality

If input quality were the only factor, we'd expect time to increase monotonically as quality drops (more rewrites needed). But we see: strong 24 min → mid 55 min → weak 42 min. The mid tier is the hardest to process, not the weak tier. This supports explanation (1) above: mid tier cached stages might just have the most complex blueprint.

## Interpretation: where's the bottleneck?

Per the plan, the Phase 4 question is: "is upstream quality or Stage 5 generation the bottleneck?"

**Answer: Stage 5 is not a pure bottleneck.**
- At the letter-grade level, Stage 5 output quality is nearly independent of input quality (all B)
- At the pass-rate level, Stage 5 output quality DOES degrade with weaker input (20pp drop)
- So Stage 5 IS robust to input quality at the "good enough to grade B" threshold, but individual file failures emerge with weaker inputs

**Practical implication**: the largest quality gains from improving upstream stages (1-4) would come from reducing individual-file failures (the 1-2 files that slip to C/D on weaker inputs), not from lifting the overall grade ceiling.

**For production**: the winner combo (qwen3.5-9b + qwen3.5-27b + mistral reviewer) produces consistent B-grade output with ~80-100% pass rate across input quality tiers. That's a defensible production configuration.

## Caveats

- **n=1 per tier** is too small to draw strong conclusions. A proper tier test would need 2-3 runs per tier.
- The tier cached stages may have a systematic bias I'm not aware of (e.g., all were generated by a particular model in the past).
- Mistral reviewer was calibrated on Phase 3 v2 blueprints; Phase 4 blueprints may differ enough to slightly shift scoring.

## Recommendation

Adopt **qwen3.5-9b + qwen3.5-27b + mistral** as the production configuration:
- Cross-family ✓ (mistral ≠ qwen)
- Consistent B grades across input quality tiers
- Cheapest compute (smallest model pair)
- Mistral reviewer has low variance and positive correlation with quality
- 100% pass rate on strong inputs, 80% on weak

If a future benchmark wants to test whether upstream improvements matter, re-run Phase 4 with n=3 per tier and compare against Phase 3 v2 cached stages for baseline noise estimate.
